# 依赖管理

依赖管理是现代软件开发中的核心实践。无论是Java项目还是前端项目，都需要有效地管理项目依赖，以确保项目的可构建性、可移植性和稳定性。

## 概述

在Java项目中，Maven和Gradle是最主流的依赖管理工具；在前端项目中，npm、yarn和pnpm则更为常见。

### 为什么需要依赖管理

1. **避免重复造轮子** - 直接使用成熟的开源库，无需自己实现
2. **版本统一** - 集中管理依赖版本，避免版本冲突
3. **传递依赖** - 自动下载依赖的依赖，减少手动管理
4. **构建可重现** - 通过锁定文件确保团队成员构建结果一致

## Maven核心概念

### 坐标

Maven坐标是唯一标识一个jar包的方式，由以下元素组成：

```xml
<groupId>org.springframework</groupId>
<artifactId>spring-core</artifactId>
<version>6.0.11</version>
<packaging>jar</packaging>
```

| 元素 | 说明 |
|------|------|
| groupId | 组织/项目组标识，通常是反向域名 |
| artifactId | 项目/模块标识 |
| version | 版本号 |
| packaging | 打包方式（jar、war、pom等） |

### 仓库

Maven仓库分为三种类型：

1. **本地仓库** - 本地缓存的jar包（`~/.m2/repository`）
2. **中央仓库** - Maven官方仓库（`https://repo.maven.apache.org/maven2`）
3. **远程仓库** - 私有仓库或镜像

```
┌─────────────────────────────────────────────────────────────┐
│                        本地仓库                             │
│                   ~/.m2/repository                          │
└─────────────────────┬───────────────────────────────────────┘
                      │ 不存在时
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                      远程仓库                               │
│         阿里云镜像 / Nexus / 私有仓库                        │
└─────────────────────┬───────────────────────────────────────┘
                      │ 不存在时
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                      中央仓库                               │
│              https://repo.maven.apache.org                  │
└─────────────────────────────────────────────────────────────┘
```

### 生命周期

Maven有三个核心生命周期：

| 生命周期 | 阶段 |
|----------|------|
| clean | pre-clean → clean → post-clean |
| default | validate → compile → test → package → install → deploy |
| site | pre-site → site → post-site → site-deploy |

**常用命令：**

```bash
mvn clean          # 清理编译文件
mvn compile        # 编译源代码
mvn test           # 运行测试
mvn package        # 打包
mvn install       # 安装到本地仓库
mvn deploy        # 部署到远程仓库
```

## pom.xml详解

### 基本结构

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- 基础信息 -->
    <groupId>com.example</groupId>
    <artifactId>my-project</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <packaging>jar</packaging>

    <name>My Project</name>
    <description>A sample project</description>

    <!-- 继承父POM -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.2.0</version>
    </parent>

    <!-- 属性配置 -->
    <properties>
        <java.version>17</java.version>
        <maven.compiler.source>17</maven.compiler.source>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <!-- 依赖管理 -->
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.example</groupId>
                <artifactId>shared-lib</artifactId>
                <version>1.0.0</version>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <!-- 项目依赖 -->
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <scope>provided</scope>
        </dependency>
    </dependencies>

    <!-- 构建配置 -->
    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

    <!-- 仓库配置 -->
    <repositories>
        <repository>
            <id>aliyun</id>
            <url>https://maven.aliyun.com/repository/public</url>
        </repository>
    </repositories>
</project>
```

### 常用元素说明

| 元素 | 说明 |
|------|------|
| modelVersion | POM模型版本 |
| parent | 继承的父项目 |
| properties | 属性变量定义 |
| dependencyManagement | 依赖版本管理 |
| dependencies | 项目依赖 |
| build/plugins | 构建插件 |
| repositories | 仓库配置 |

## 依赖管理

### dependencies vs dependencyManagement

**dependencies**

所有声明在`<dependencies>`中的依赖会自动添加到项目中。如果子项目不声明依赖，会从父项目继承。

```xml
<!-- 父POM -->
<dependencies>
    <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-core</artifactId>
        <version>6.0.11</version>
    </dependency>
</dependencies>

<!-- 子POM 自动继承 spring-core -->
```

> `<dependencies>`中的jar直接加到项目中，管理的是依赖关系。

**dependencyManagement**

`<dependencyManagement>`只是声明依赖，并不实际引入。主要用于集中管理依赖版本。

```xml
<!-- 父POM -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.example</groupId>
            <artifactId>shared-lib</artifactId>
            <version>1.0.0</version>
        </dependency>
    </dependencies>
</dependencyManagement>

<!-- 子POM 需要显式声明才会引入 -->
<dependencies>
    <dependency>
        <groupId>org.example</groupId>
        <artifactId>shared-lib</artifactId>
        <!-- 不写version，会使用父POM中声明的版本 -->
    </dependency>
</dependencies>
```

> `<dependencyManagement>`主要管理版本，不添加依赖关系。子pom不一定要继承父pom所定义的版本。

### 版本确定规则

1. 如果dependencies中的dependency声明了version，以声明的版本为准
2. 如果没有声明version，会到dependencyManagement中查找
3. 如果dependencyManagement中也没有，会报错

## 依赖范围

依赖范围（scope）决定了依赖在什么阶段可用：

| scope | 编译 | 测试 | 运行 | 典型场景 |
|-------|------|------|------|----------|
| compile | ✓ | ✓ | ✓ | 默认，大部分jar |
| provided | ✓ | ✓ | ✗ | servlet-api（容器提供） |
| runtime | ✗ | ✓ | ✓ | JDBC驱动 |
| test | ✗ | ✓ | ✗ | JUnit、Mockito |
| system | ✓ | ✓ | ✗ | 本地jar（不推荐） |
| import | - | - | - | 导入依赖管理 |

### 使用示例

```xml
<!-- 编译和运行时都需要 -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-core</artifactId>
</dependency>

<!-- 由容器提供，不需要打包 -->
<dependency>
    <groupId>jakarta.servlet</groupId>
    <artifactId>jakarta.servlet-api</artifactId>
    <scope>provided</scope>
</dependency>

<!-- 仅测试时需要 -->
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <scope>test</scope>
</dependency>

<!-- 运行时不需编译，测试时需要 -->
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>
```

## 可传递性依赖

### 依赖传递机制

Maven会自动解析传递依赖。假设：
- A依赖B
- B依赖C

那么A会自动获得C的依赖。

```bash
# 查看项目依赖树
mvn dependency:tree

# 查看特定依赖
mvn dependency:tree -Dincludes=org.springframework
```

### 依赖调解

当多个传递依赖指向不同版本的同一jar时，Maven使用以下原则：

1. **路径优先** - 选择依赖路径最短的版本
2. **声明优先** - 路径相同时，选择先声明的版本

```xml
<!-- 强制指定版本（不建议） -->
<dependency>
    <groupId>org.example</groupId>
    <artifactId>lib</artifactId>
    <version>2.0.0</version>
</dependency>
```

## 依赖冲突与解决

### 冲突表现

- 编译失败
- 运行时异常（如NoSuchMethodError）
- 行为异常

### 解决方案

**1. 排除传递依赖**

```xml
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-core</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.example</groupId>
            <artifactId>conflicting-lib</artifactId>
        </exclusion>
    </exclusions>
</dependency>
```

**2. 锁定版本**

在dependencyManagement中显式声明版本：

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.example</groupId>
            <artifactId>lib</artifactId>
            <version>1.0.0</version>
        </dependency>
    </dependencies>
</dependencyManagement>
```

**3. 使用maven-enforcer-plugin**

```xml
<plugin>
    <groupId>org.codehaus.mojo</groupId>
    <artifactId>enforcer-maven-plugin</artifactId>
    <executions>
        <execution>
            <id>enforce-dependency-convergence</id>
            <goals>
                <goal>enforce</goal>
            </goals>
            <configuration>
                <rules>
                    <dependencyConvergence/>
                    <requireNoRepositories>
                        <banRepositories>true</banRepositories>
                    </requireNoRepositories>
                </rules>
            </configuration>
        </execution>
    </executions>
</plugin>
```

## 多模块项目

### 聚合工程

父项目管理多个子模块：

```xml
<!-- 父POM -->
<project>
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>parent-project</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>

    <!-- 聚合子模块 -->
    <modules>
        <module>module-a</module>
        <module>module-b</module>
        <module>module-common</module>
    </modules>
</project>
```

### 继承关系

子模块继承父项目：

```xml
<!-- 子模块 -->
<project>
    <parent>
        <groupId>com.example</groupId>
        <artifactId>parent-project</artifactId>
        <version>1.0.0</version>
    </parent>
    <artifactId>module-a</artifactId>
</project>
```

### 完整示例

```
parent-project/
├── pom.xml              # 聚合 + 父POM
├── module-common/
│   └── pom.xml
├── module-dao/
│   └── pom.xml
├── module-service/
│   └── pom.xml
└── module-web/
    └── pom.xml
```

**父POM配置：**

```xml
<project>
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>parent-project</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>

    <modules>
        <module>module-common</module>
        <module>module-dao</module>
        <module>module-service</module>
        <module>module-web</module>
    </modules>

    <!-- 统一版本管理 -->
    <properties>
        <spring.version>6.0.11</spring.version>
        <java.version>17</java.version>
    </properties>

    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework</groupId>
                <artifactId>spring-core</artifactId>
                <version>${spring.version}</version>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <!-- 统一插件管理 -->
    <build>
        <pluginManagement>
            <plugins>
                <plugin>
                    <groupId>org.apache.maven.plugins</groupId>
                    <artifactId>maven-compiler-plugin</artifactId>
                    <version>3.11.0</version>
                    <configuration>
                        <source>${java.version}</source>
                        <target>${java.version}</target>
                    </configuration>
                </plugin>
            </plugins>
        </pluginManagement>
    </build>
</project>
```

## 私有仓库

### 阿里云镜像

```xml
<!-- pom.xml 配置镜像 -->
<repositories>
    <repository>
        <id>aliyun</id>
        <name>Aliyun Maven Repository</name>
        <url>https://maven.aliyun.com/repository/public</url>
        <releases>
            <enabled>true</enabled>
        </releases>
        <snapshots>
            <enabled>false</enabled>
        </snapshots>
    </repository>
</repositories>

<mirrors>
    <mirror>
        <id>aliyun-maven</id>
        <name>Aliyun Maven Mirror</name>
        <url>https://maven.aliyun.com/repository/public</url>
        <mirrorOf>central</mirrorOf>
    </mirror>
</mirrors>
```

或修改`~/.m2/settings.xml`：

```xml
<mirrors>
    <mirror>
        <id>aliyun-maven</id>
        <mirrorOf>*</mirrorOf>
        <name>Aliyun Maven Mirror</name>
        <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```

            <updatePolicy>always</updatePolicy>

    </repository>
</repositories>

## 前端依赖管理

### npm

```bash
# 初始化项目
npm init

# 安装依赖
npm install lodash

# 保存到生产依赖
npm install express --save

# 保存到开发依赖
npm install webpack --save-dev

# 全局安装
npm install -g typescript

# 安装项目所有依赖
npm install

# 查看依赖
npm list
npm list --depth=0

# 更新依赖
npm update lodash

# 删除依赖
npm uninstall lodash
```

### yarn

```bash
# 初始化
yarn init

# 安装
yarn add lodash
yarn add webpack --dev

# 全局
yarn global add typescript

# 安装所有
yarn install

# 更新
yarn upgrade lodash

# 删除
yarn remove lodash
```

### pnpm

```bash
# 安装
pnpm add lodash
pnpm add -D webpack

# 安装所有
pnpm install

# 更新
pnpm update lodash
```

### 对比

| 特性 | npm | yarn | pnpm |
|------|-----|------|------|
| 速度 | 较慢 | 快 | 最快 |
| 磁盘占用 | 大 | 较大 | 小 |
| 依赖结构 | 扁平 | 扁平 | 符号链接 |
| lock文件 | package-lock.json | yarn.lock | pnpm-lock.yaml |
| 兼容性 | 好 | 好 | 较好 |

> **推荐：** pnpm对磁盘空间最友好，且使用符号链接避免重复依赖。

### package-lock.json

锁定依赖版本，确保团队成员安装相同版本的依赖：

```bash
# 生成lock文件
npm install   # 会生成 package-lock.json

# 忽略lock文件重新安装
npm install --no-package-lock

# 使用lock文件安装
npm ci  # 从lock文件安装，更快且更可靠
```

## 版本管理策略

### 语义化版本（SemVer）

版本格式：`主版本.次版本.修订号`

```bash
1.2.3
│ │ │
│ │ └── 修订号：bug修复，兼容旧版
│ └──── 次版本：新功能，兼容旧版
└────── 主版本：不兼容的API变更
```

**版本范围：**

```xml
<!-- 精确版本 -->
<version>1.0.0</version>

<!-- 范围版本 -->
<version>[1.0.0,2.0.0)</version>  <!-- >=1.0.0 且 <2.0.0 -->

<!-- 动态版本 -->
<version>1.0.x</version>   <!-- 1.0.0, 1.0.1, 1.0.2 -->
<version>1.x</version>     <!-- 1.0.0 ~ 1.9.9 -->
<version>RELEASE</version> <!-- 最新发布版 -->
<version>LATEST</version>  <!-- 最新版本 -->
```

### 锁定版本策略

**不推荐：**

```xml
<!-- 生产环境不应使用动态版本 -->
<version>LATEST</version>
<version>1.x</version>
```

**推荐：**

```xml
<!-- 使用固定版本 -->
<version>1.2.3</version>

<!-- 或使用版本范围，但要有上下限 -->
<version>[1.0.0,2.0.0)</version>
```

### 依赖升级原则

1. **小版本升级** - 定期进行，关注变更日志
2. **大版本升级** - 谨慎评估，做好回滚计划
3. **安全补丁** - 及时跟进
4. **锁定传递依赖** - 防止间接依赖升级导致问题

## 常见问题FAQ

### Q: 依赖下载失败？

A: 检查网络和镜像配置
```bash
# 查看详细错误
mvn dependency:resolve -X

# 检查本地仓库
ls ~/.m2/repository/

# 强制更新
mvn dependency:purge-local-repository -DreResolve=true
```

### Q: 依赖打不进去jar包？

A: 检查打包插件配置
```xml
<build>
    <resources>
        <resource>
            <directory>src/main/resources</directory>
        </resource>
    </resources>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>
```

### Q: 依赖冲突导致NoSuchMethodError？

A: 使用依赖树分析并排除
```bash
mvn dependency:tree | grep conflicting
```

### Q: 镜像不生效？

A: 检查settings.xml配置优先级
```bash
# 查看生效的配置
mvn help:effective-settings

# 指定settings文件
mvn -s /path/to/settings.xml install
```

### Q: 私有仓库认证失败？

A: 配置认证信息
```xml
<!-- settings.xml -->
<servers>
    <server>
        <id>nexus-releases</id>
        <username>admin</username>
        <password>password</password>
    </server>
</servers>
```

### Q: Maven构建慢？

A: 优化配置
```xml
<!-- 本地仓库并行下载 -->
<settings>
    <mirrors>
        <mirror>
            <id>aliyun</id>
            <url>https://maven.aliyun.com/repository/public</url>
            <mirrorOf>*</mirrorOf>
        </mirror>
    </mirrors>
    <profiles>
        <profile>
            <id>jdk-17</id>
            <activation>
                <activeByDefault>true</activeByDefault>
                <jdk>17</jdk>
            </activation>
            <properties>
                <maven.compiler.source>17</maven.compiler.source>
                <maven.compiler.target>17</maven.compiler.target>
            </properties>
        </profile>
    </profiles>
</settings>
```

### Q: npm安装依赖失败？

A: 常见解决方案
```bash
# 清理缓存
npm cache clean --force

# 删除node_modules重新安装
rm -rf node_modules
npm install

# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 查看问题
npm install --verbose
```

### Q: package.json和package-lock.json不一致？

A: 使用npm ci
```bash
# 使用lock文件安装，确保一致
npm ci

# 或者删除lock文件重新生成
rm package-lock.json
npm install
```
