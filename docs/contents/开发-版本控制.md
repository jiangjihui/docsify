# 版本控制

版本控制系统（Version Control System）是现代软件开发中不可或缺的工具。它记录文件内容的变化，以便将来查阅特定版本的改动历史。

## 概述

### 什么是版本控制

版本控制系统是一种记录若干文件内容变化，以便将来查阅特定版本修订情况的系统。它的主要功能包括：

- **历史记录** - 记录每次修改的内容、时间、作者
- **版本对比** - 查看任意两个版本之间的差异
- **回退功能** - 将项目回退到之前的某个状态
- **分支开发** - 在独立分支上开发功能，互不影响
- **协作支持** - 多人协同开发，合并代码

### Git简介

Git是由Linus Torvalds于2005年创建的分布式版本控制系统。与传统的集中式版本控制（如SVN）不同，Git是分布式的，每个开发者都拥有完整的仓库副本。

**Git的特点：**

- **分布式** - 每个开发者都有完整仓库，无需联网也能提交和查看历史
- **本地分支** - 创建和切换分支非常快速
- **数据完整性** - 使用SHA-1哈希确保数据完整性
- **开源免费** - 开源项目，可免费使用

### 集中式 vs 分布式

| 特性 | 集中式（SVN） | 分布式（Git） |
|------|---------------|---------------|
| 仓库位置 | 中央服务器 | 每个客户端都有完整仓库 |
| 网络依赖 | 必须联网 | 可离线工作 |
| 分支效率 | 较慢 | 非常快 |
| 存储效率 | 较低 | 高 |

### 常用代码托管平台

- **GitHub** - 全球最大的开源代码托管平台
- **GitLab** - 提供自托管版本，支持DevOps
- **Gitee（码云）** - 国内代码托管平台
- **Bitbucket** - 支持Git和Mercurial

## Git基础

### 安装和配置

```bash
# 配置用户名和邮箱（必须）
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 设置默认分支名
git config --global init.defaultBranch master

# 设置合并工具
git config --global merge.tool vimdiff

# 查看配置
git config --list
git config user.name  # 查看特定配置
```

### 仓库创建

```bash
# 初始化新仓库
git init

# 克隆现有仓库
git clone https://github.com/user/repo.git
git clone https://github.com/user/repo.git my-folder  # 指定目录名
```

### 工作区域

Git有三个主要区域：

```
┌─────────────────────────────────────────────────────────┐
│                    工作目录 (Working Directory)         │
│              你正在编辑的文件所在的位置                   │
└────────────────────────┬────────────────────────────────┘
                         │ git add
                         ▼
┌─────────────────────────────────────────────────────────┐
│                   暂存区 (Staging Area)                  │
│              准备提交的文件列表                          │
└────────────────────────┬────────────────────────────────┘
                         │ git commit
                         ▼
┌─────────────────────────────────────────────────────────┐
│                  Git仓库 (Repository)                   │
│              保存项目历史的地方                          │
└─────────────────────────────────────────────────────────┘
```

### .gitignore

指定Git忽略的文件和目录：

```bash
# 忽略所有 .log 文件
*.log

# 忽略 node_modules 目录
node_modules/

# 忽略特定目录中的所有文件
logs/*
!logs/.gitkeep  # 但保留 .gitkeep

# 忽略特定文件
secret.yaml

# 忽略以 ~ 结尾的文件
*~

# 注释
# 这是注释

# 忽略所有 target 目录
target/
```

## 常用命令

### 基础命令

```bash
# 查看当前状态
git status

# 查看变更详情
git diff
git diff --staged  # 暂存区的变更

# 添加文件到暂存区
git add filename.txt
git add .          # 添加所有修改
git add -p         # 交互式添加（分块添加）

# 提交更改
git commit -m "提交说明"
git commit -am "添加并提交"  # 仅对已跟踪文件有效

# 查看提交历史
git log
git log --oneline           # 简洁格式
git log -n 5                # 最近5条
git log --graph             # 图形化显示
git log --author="username" # 按作者筛选
```

### 远程命令

```bash
# 查看远程仓库
git remote -v

# 添加远程仓库
git remote add origin https://github.com/user/repo.git

# 移除远程仓库
git remote remove origin

# 抓取远程更新（不合并）
git fetch origin

# 拉取并合并
git pull origin main
git pull --rebase origin main  # 使用rebase合并

# 推送到远程
git push origin main
git push -u origin main  # 推送并设置上游分支
git push origin --delete branch  # 删除远程分支
```

### 查看命令

```bash
# 查看某次提交的内容
git show commit-id

# 查看文件的修改历史
git log filename
git log -p filename

# 查看文件某次修改的内容
git log -1 -p filename

# 查看文件的每行最后修改
git blame filename

# 查看两个分支的差异
git diff branch1 branch2
git diff branch1..branch2  # 同上
git diff branch1...branch2 # 三点表示共同祖先的差异
```

## 分支管理

### 基本操作

```bash
# 查看分支
git branch           # 本地分支
git branch -r        # 远程分支
git branch -a        # 所有分支

# 创建分支（不切换）
git branch feature-new

# 切换分支
git checkout feature-new
git switch feature-new  # Git 2.23+

# 创建并切换
git checkout -b feature-new
git switch -c feature-new  # Git 2.23+

# 删除分支
git branch -d feature-new      # 已合并才能删除
git branch -D feature-new       # 强制删除
git push origin --delete feature-new  # 删除远程分支
```

### 合并分支

```bash
# 合并分支到当前分支
git merge feature-new

# 使用rebase合并（保持线性历史）
git rebase main

# 解决冲突后继续rebase
git rebase --continue
git rebase --abort  # 放弃rebase
```

### 变基（rebase）vs 合并（merge）

```
merge 结果：                    rebase 结果：
  A---B---C (main)                A---B---C---D---E (main)
       \                           \
        D---E---F (feature)         D---E---F (feature)
```

- **merge**：保留完整历史，直观但分支线复杂
- **rebase**：线性历史，清晰但会修改历史

> **注意**：不要对已推送的公共分支执行rebase

## 版本回退

### git reset

```bash
# 回退到指定提交（默认mixed）
git reset commit-id

# 三种模式：
git reset --soft commit-id    # 保留暂存区和工作区
git reset --mixed commit-id   # 保留工作区（默认）
git reset --hard commit-id    # 丢弃所有修改（危险！）

# 常用操作：
git reset --hard HEAD~1      # 回退到上一个提交
git reset --hard origin/main  # 回退到远程最新
```

**模式对比：**

| 模式 | 暂存区 | 工作区 | 说明 |
|------|--------|--------|------|
| --soft | 保留 | 保留 | 回到暂存区 |
| --mixed | 清空 | 保留 | 回到工作区（默认） |
| --hard | 清空 | 清空 | 完全丢弃（危险） |

### git revert

创建新提交来撤销指定提交的变更：

```bash
# 撤销最近一次提交
git revert HEAD

# 撤销指定提交
git revert commit-id

# 撤销多个提交（依次撤销）
git revert commit-id1..commit-id3

# 撤销但不自动提交
git revert -n commit-id
```

**reset vs revert：**

- **reset**：修改历史，会"丢失"某些提交
- **revert**：新增一个撤销提交，安全的公共分支操作

### git reflog

恢复误操作的神器：

```bash
# 查看所有操作历史
git reflog

# 输出格式：
# a1b2c3d HEAD@{0}: commit: 提交说明
# d4e5f6g HEAD@{1}: checkout: moving from main to feature

# 恢复到指定操作
git reset --hard HEAD@{2}
```

> **注意**：`reflog`只保留最近90天的记录

### IDEA中的版本回退

1. 在需要回滚的那次提交记录上右键：`Reset Current Branch to Here...`
2. 选择模式（Hard/Mixed/Soft）
3. 强制推送到远程：
   ```bash
   git push origin HEAD --force
   ```
   或者在IDEA的Git设置中将主分支（master）保护去掉，然后在推送时选择Force push

> **警告**：强制推送会覆盖远程历史，其他协作者需要重新同步。

## 远程仓库

### 基础操作

```bash
# 查看远程仓库
git remote -v

# 添加远程仓库
git remote add origin https://github.com/user/repo.git
git remote add upstream https://github.com/original/repo.git  # 上游仓库

# 查看远程仓库详情
git remote show origin

# 重命名远程仓库
git remote rename origin old-origin

# 移除远程仓库
git remote remove origin
```

### 克隆和跟踪

```bash
# 克隆仓库
git clone https://github.com/user/repo.git

# 克隆指定分支
git clone -b develop https://github.com/user/repo.git

# 克隆指定分支并设置名称
git clone -b develop https://github.com/user/repo.git my-project

# 添加上游跟踪
git branch --set-upstream-to=origin/main main
git branch -u origin/main main  # 简写
```

### fetch vs pull

```bash
# fetch：下载远程更新，但不合并
git fetch origin

# pull：下载并合并
git pull origin main

# pull with rebase
git pull --rebase origin main

# 使用rebase保持线性历史
git pull --rebase
```

## 标签管理

### 创建标签

```bash
# 创建轻量标签
```

### 管理和推送

```bash
# 查看标签

# 查看标签详情

git tag -d v1.0.0

# 删除远程标签
git push origin --delete v1.0.0
```

## 协作流程

### Git Flow工作流

```
main/master ────●────────────────●─────────────── (生产分支)
                ↑                  ↑
                │                  │
                │    release/1.0   │
                └──────────────────┘
                               │
develop ────●────●────●────●───┘ (开发分支)
                ↑   ↑   ↑   ↑
                │   │   │   │
            feature/  │   │   (功能分支)
                ──────┘   │
                          │
                      hotfix/   (热修复分支)
                      ─────────
```

**分支说明：**

| 分支 | 用途 | 命名规则 |
|------|------|----------|
| main/master | 生产环境 | - |
| develop | 开发环境 | - |
| feature/* | 新功能 | feature/功能名 |
| release/* | 发布准备 | release/版本号 |
| hotfix/* | 紧急修复 | hotfix/问题描述 |

### Fork和PR流程

1. **Fork**：在GitHub页面点击Fork按钮，复制仓库到自己的账户
2. **Clone**：克隆自己Fork的仓库到本地
3. **添加上游仓库**：
   ```bash
   git remote add upstream 原仓库地址
   ```
4. **创建功能分支**：
   ```bash
   git checkout -b feature-new
   ```
5. **开发并提交**
6. **同步上游更新**：
   ```bash
   git fetch upstream
   git rebase upstream/main
   ```
7. **推送并创建PR**

### 代码审查要点

- **代码风格** - 是否符合项目规范
- **功能实现** - 是否满足需求
- **测试覆盖** - 是否有足够的测试
- **安全性** - 是否有安全漏洞
- **性能** - 是否有性能问题
- **可读性** - 代码是否易读

## IDEA集成

### 基本配置

1. `Settings` → `Version Control` → `Git`
2. 配置Git执行文件路径
3. 测试连接

### 常用操作

**提交：**
1. 修改文件后，底部会显示`Git: *changed files`
2. 点击打开Version Control面板
3. 右键文件 → `Add`（添加到暂存区）
4. 右键文件 → `Commit`（提交）
5. 填写提交信息，点击Commit

**切换分支：**
- 右键项目 → `Git` → `Branches` → 选择分支

**查看历史：**
- 右键文件 → `Git` → `Show History`
- 或使用 `View` → `Tool Windows` → `Version Control`

**解决冲突：**
1. 合并时出现冲突会提示
2. 点击 `Merge` 打开合并工具
3. 手动选择或修改冲突内容
4. 标记为 resolved 后提交

## 常见问题FAQ

### Q: 合并冲突怎么办？

A: 手动解决后重新提交
```bash
# 查看冲突文件
git status

# 打开冲突文件，保留需要的内容
# 冲突标记：
# <<<<<<< HEAD
# 当前分支内容
# =======
# 要合并的内容
# >>>>>>> branch-name

# 解决后
git add filename
git commit -m "解决冲突"
```

### Q: 误删分支如何恢复？

A: 使用reflog
```bash
git reflog
git checkout -b branch-name commit-id
```

### Q: 强制推送失败？

A: 可能远程有更新，先拉取再推送
```bash
git pull --rebase origin branch-name
git push origin branch-name
```

### Q: 如何撤销最后一次提交？

A:
```bash
# 保留修改（在暂存区）
git reset --soft HEAD~1

# 保留修改（在工作区）
git reset --mixed HEAD~1

# 丢弃修改
git reset --hard HEAD~1
```

### Q: 远程仓库有更新，本地想保持干净？

A:
```bash
# 方式1：stash
git stash
git pull
git stash pop

# 方式2：reset
git reset --hard origin/main
```

### Q: .gitignore不生效？

A: 已经跟踪的文件需要先移除
```bash
# 移除缓存
git rm -r --cached .
git add .
git commit -m "更新gitignore"
```

### Q: 提交信息写错了怎么办？

A: 修改最后一次提交
```bash
git commit --amend -m "正确的提交信息"
```

### Q: 子模块管理？

A:
```bash
# 添加子模块
git submodule add https://github.com/user/repo.git path/to/repo

# 克隆包含子模块的项目
git clone --recursive https://github.com/user/repo.git

# 更新子模块
git submodule update --init --recursive
```
